Simple SF Energy Sharing ABM

Simulation time is explicitly 2025-01-01 00:00 through 2025-01-30 23:00, with no-sun disturbance on 2025-01-15.

Agent state: A_i(t) = {building_type, norm, generation_i(t), demand_i(t), storage_i(t), health_i(t)}. This version uses only residential buildings.

Load profile source: local SF residential rows from energy_profiles_hourly_used.csv, joined to residential buildings in building_energy_metadata.csv by profile_id. The GitHub repo stores the compact prepared version in data/agents_initial.json.

Solar generation potential source: cached NASA POWER 2025 hourly ALLSKY_SFC_SW_DWN for San Francisco, normalized to [0, 1]. Generation uses generation_i(t) = normalized_solar(t) * max(demand_i).

Battery/storage size source: no external battery-size data is used. storage_i(t) is only carried-over unused surplus energy from the previous hour; the minimal model has no battery capacity parameter.

Building health is continuous: health_i(t) = clip((generation + starting_storage + energy_received - energy_exported) / demand, 0, 1). A building is dead only when health_i(t) = 0.

Energy sharing rule: gift = min(surplus, energy_request) * norm_donor. The two scenarios set every residential building's norm_i to 0 or 1.

Only two performance metrics are reported: % building alive and resilience. Here Q(t) is system performance, defined as the average building health: Q(t) = mean_i health_i(t). Resilience is normalized area under that curve: R = integral Q(t) dt / integral Q0 dt, where Q0 = 1.
